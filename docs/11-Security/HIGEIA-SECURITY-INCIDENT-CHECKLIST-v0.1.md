# Higeia — Security Incident Initial Checklist

**Versão:** 0.1  
**Data:** 19 de julho de 2026  

## Ação imediata

- [ ] Registrar data e hora da descoberta.
- [ ] Abrir identificador do incidente.
- [ ] Preservar evidências.
- [ ] Conter o acesso ou vazamento.
- [ ] Não apagar logs.
- [ ] Identificar sistemas afetados.
- [ ] Identificar categorias de dados.
- [ ] Identificar dados sensíveis.
- [ ] Estimar titulares afetados.
- [ ] Avaliar risco ou dano relevante.
- [ ] Acionar responsável técnico.
- [ ] Acionar responsável jurídico/privacidade.
- [ ] Registrar decisões.
- [ ] Avaliar comunicação à ANPD.
- [ ] Avaliar comunicação aos titulares.
- [ ] Corrigir a causa.
- [ ] Monitorar recorrência.
- [ ] Emitir relatório pós-incidente.

## Informações mínimas

- natureza do incidente;
- data do conhecimento;
- dados afetados;
- titulares;
- medidas de segurança;
- riscos;
- contenção;
- comunicação;
- responsável;
- aprendizados.

## Observação

A decisão de comunicação deve seguir a legislação e o regulamento vigente, com apoio jurídico.
