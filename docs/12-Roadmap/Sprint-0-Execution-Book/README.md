## Uso

O Execution Book define o plano.

O backlog organiza épicos e histórias.

O template padroniza histórias.

A task matrix detalha tarefas.

O environment plan define ambientes.

O repository standard orienta Git e código.

O CI/CD checklist controla releases.

O security baseline define controles mínimos.

O AI checklist prepara a plataforma de IA.

O observability plan prepara operação e custos.

O rollback runbook prepara falhas.

A readiness review decide se o Sprint 1 pode começar.

## Regra

Sprint 0 termina por evidência e readiness, não por prazo decorrido.
