# Higeia — Product Requirements Baseline

## Functional

- Professional can create account.
- Professional can invite patient.
- Patient can consent.
- Patient can complete anamnesis.
- Patient can submit check-in.
- Professional can review patient data.
- System can generate structured summary.
- Professional can correct or approve summary.
- System can display longitudinal history.
- System can log critical actions.

## Non-functional

- Data isolation.
- Access control.
- Auditability.
- Availability target.
- Backup.
- Rollback.
- Monitoring.
- Privacy controls.
- Explainability labels.
- Controlled cost.

## Constraints

- No autonomous diagnosis.
- No autonomous prescription.
- No use outside approved scope.
- Human review required for sensitive outputs.
- Alpha user limits enforced.
