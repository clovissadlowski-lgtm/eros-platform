Esta pasta concentra a governança do ciclo de vida de modelos, prompts, schemas, guardrails e configurações de IA do Higeia.

## Uso

O framework define o ciclo de vida.

O change record documenta cada alteração.

O deployment record registra a implantação.

O rollback runbook orienta a reversão.

O model card descreve o modelo.

O release gate controla aprovações.

As release notes comunicam mudanças.

O deprecation plan organiza retirada.

## Regra

Nenhuma mudança relevante deve chegar à produção sem versão, avaliação, aprovação, rollout controlado, monitoramento e rollback testado.
