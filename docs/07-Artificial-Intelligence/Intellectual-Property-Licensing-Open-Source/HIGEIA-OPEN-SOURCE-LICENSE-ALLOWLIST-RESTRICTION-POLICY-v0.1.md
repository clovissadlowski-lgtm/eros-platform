# Higeia — Open Source License Allowlist and Restriction Policy

## Approved by default

Examples may include permissive licenses after verification.

## Restricted

Licenses requiring case-by-case review, including:

- copyleft;
- network copyleft;
- source-available;
- custom;
- attribution-heavy;
- patent-restrictive.

## Prohibited by default

- unknown license;
- noncommercial;
- research-only for commercial use;
- no-derivatives when modification is needed;
- license incompatible with intended distribution.

## Rule

The actual license text and version always prevail over category assumptions.
