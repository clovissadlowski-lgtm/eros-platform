## Uso

O framework define a política geral.

A BIA mede impacto.

O service register inventaria serviços e dependências.

O DR plan organiza recuperação ampla.

A fallback matrix define modos degradados.

O backup standard define cópia e restauração.

O runbook guia recuperação por serviço.

O exercise plan organiza simulações.

O test record registra resultados.

O communication template prepara mensagens.

O readiness checklist controla prontidão.

## Regra

Continuidade segura pode exigir redução de funcionalidade, suspensão de IA ou operação degradada. Nenhum backup deve ser considerado confiável sem teste de restauração.
