# Higeia — Inclusive Design Review Checklist

## Language

- [ ] No stigmatizing language.
- [ ] No moral judgment.
- [ ] Technical terms explained.
- [ ] Reading level appropriate.
- [ ] Regional expressions considered.
- [ ] Calls to action are clear.

## Socioeconomic context

- [ ] Cost considered.
- [ ] Availability considered.
- [ ] Time constraints considered.
- [ ] Equipment assumptions reviewed.
- [ ] Transport and access considered.
- [ ] Alternatives offered.

## Digital access

- [ ] Works on small screens.
- [ ] Works on slower connections.
- [ ] Sessions can resume.
- [ ] Data usage is reasonable.
- [ ] No unnecessary download.
- [ ] Critical content remains accessible.

## Accessibility

- [ ] Keyboard navigation.
- [ ] Screen reader support.
- [ ] Contrast.
- [ ] Focus order.
- [ ] Labels.
- [ ] Error messages.
- [ ] Zoom.
- [ ] Non-visual alternatives.

## Control

- [ ] User can correct.
- [ ] User can dispute.
- [ ] Human review available when needed.
- [ ] No dark pattern.
