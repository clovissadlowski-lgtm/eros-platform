# Higeia — AI Incident Response Playbook

## First 15 minutes

- [ ] Open incident record.
- [ ] Assign Incident ID.
- [ ] Classify category.
- [ ] Estimate severity.
- [ ] Preserve evidence.
- [ ] Activate feature flag or containment.
- [ ] Notify core owners.
- [ ] Check whether users remain exposed.
- [ ] Decide immediate rollback.

## First hour

- [ ] Confirm scope.
- [ ] Identify affected versions.
- [ ] Review logs and metrics.
- [ ] Evaluate privacy and safety.
- [ ] Activate fallback.
- [ ] Notify clinical/legal reviewers if required.
- [ ] Define communication owner.
- [ ] Create working timeline.

## Investigation

- [ ] Reproduce.
- [ ] Identify root cause.
- [ ] Identify contributing factors.
- [ ] Search for similar cases.
- [ ] Assess historical impact.
- [ ] Define correction.
- [ ] Create regression test.

## Recovery

- [ ] Validate fix.
- [ ] Run evaluation.
- [ ] Run safety and privacy tests.
- [ ] Confirm monitoring.
- [ ] Approve controlled rollout.
- [ ] Observe.
- [ ] Close or reopen.

## Mandatory P0/P1 outputs

- [ ] Incident report.
- [ ] Postmortem.
- [ ] Regression case.
- [ ] Corrective actions.
- [ ] Governance review.
- [ ] Communication decision.
