# AI Incident Management

Esta pasta concentra a gestão de incidentes de IA e o risco de modelos do Higeia.

## Documentos

- `HIGEIA-AI-INCIDENT-MANAGEMENT-MODEL-RISK-FRAMEWORK-v0.1.md`
- `HIGEIA-AI-INCIDENT-REPORT-TEMPLATE-v0.1.md`
- `HIGEIA-MODEL-RISK-REGISTER-TEMPLATE-v0.1.md`
- `HIGEIA-AI-MODEL-INVENTORY-TEMPLATE-v0.1.md`
- `HIGEIA-AI-INCIDENT-POSTMORTEM-TEMPLATE-v0.1.md`
- `HIGEIA-AI-INCIDENT-RESPONSE-PLAYBOOK-v0.1.md`

## Uso

O framework define a política geral.

O incident report registra cada ocorrência.

O risk register organiza riscos conhecidos.

O model inventory mantém os modelos aprovados.

O postmortem documenta causas e aprendizados.

O playbook orienta a resposta operacional.

## Regra

Todo incidente relevante deve gerar contenção, registro, análise, correção, teste de regressão e monitoramento.
