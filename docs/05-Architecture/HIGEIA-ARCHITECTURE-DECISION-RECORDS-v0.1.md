# Higeia — Architecture Decision Records (ADR)

Versão: 0.1

## Objetivo

Padronizar como decisões técnicas serão registradas durante o desenvolvimento.

Cada ADR deve conter:

1. Contexto
2. Problema
3. Opções consideradas
4. Decisão adotada
5. Consequências
6. Alternativas rejeitadas
7. Critérios para revisão futura

## Modelo

```md
# ADR-0001 - Título

Status: Proposto | Aceito | Substituído

## Contexto

...

## Problema

...

## Opções

A.
B.
C.

## Decisão

...

## Justificativa

...

## Consequências

Positivas:
- ...

Negativas:
- ...

## Revisão

Quando esta decisão deverá ser reavaliada.
```

## ADRs planejados

- ADR-0001 — Estrutura do repositório
- ADR-0002 — Next.js como framework principal
- ADR-0003 — Supabase como backend inicial
- ADR-0004 — Prisma como ORM
- ADR-0005 — Estratégia de autenticação
- ADR-0006 — Modelo HIE
- ADR-0007 — Observabilidade
- ADR-0008 — Estratégia de IA

## Regras

- Uma decisão importante = um ADR.
- ADRs nunca são apagados; podem ser substituídos por novos.
- Cada ADR recebe numeração sequencial.
